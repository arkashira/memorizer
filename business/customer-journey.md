## **Aware**
- **Trigger Event**: Developer encounters performance issues (slow app, crashes) but lacks tools to diagnose memory misuse.

- **Delight Opportunity**: Highlight specific, actionable insights (e.g., "Identified 3 memory leaks in your React app in 2 minutes").
- **Metric**: # of users who sign up after experiencing performance issues.

## **Consider**
- **Trigger Event**: Developer researches solutions (e.g., Google "memory optimization tools for Python").
- **Friction**: Skepticism about tool efficacy; fear of integration complexity with existing workflows.
- **Emotions**: Cautious optimism, comparison fatigue.
- **Delight Opportunity**: Demo video showing seamless integration with VS Code/CLI + side-by-side comparison vs. competitors (e.g., "50% faster analysis than Valgrind").
- **Metric**: # of demo sign-ups or free trial starts.

## **Try**
- **Trigger Event**: Downloads trial or uses free tier (e.g., "Analyze 1 repo for free").
- **Friction**: Setup friction (e.g., API keys, CLI configuration) or unclear initial results.
- **Emotions**: Hope → frustration → relief (if results are clear).
- **Delight Opportunity**: "First analysis complete in <2 mins" + prioritized recommendations (e.g., "Fix this 1-line change to reduce memory by 30%").
- **Metric**: Trial-to-paid conversion rate; time-to-first-analysis (<2 mins target).

## **Adopt**
- **Trigger Event**: Regular use for codebase maintenance (e.g., pre-deploy checks).

- **Emotions**: Satisfaction → dependency anxiety (e.g., "What if it breaks in production?").
- **Delight Opportunity**: Proactive alerts ("Memory usage spiked 20% in your latest commit") + SLA guarantees.
- **Metric**: 90-day retention rate; # of monthly active repos analyzed.

## **Expand**
- **Trigger Event**: Team adoption or need for enterprise features (e.g., SSO, audit logs).
- **Friction**: Cost concerns; onboarding friction for large teams.
- **Emotions**: Enthusiasm → cautious negotiation.
